Task for Senior Javascript Developer with Angular

You should develop a simple project management web app with the following functionalities and requirements. Full implementation is not required, but will be considered a big advantage. You should follow JS and Angular’s best practices and use ES6 as much as possible. There are no limitations about the design other than that it should be comprehensible and user-friendly. You may use CSS & JS frameworks and libraries including, but not limited to, Bootstrap, MaterializeCSS(recommended), MDL(recommended),SemanticUI, AngularUI, jQuery(not recommended), underscore, RxJS, or any other tool such as npm, bower, Babel

Functionalities:

  • The app should use angular’s routing with the following routes:
    • Dashboard - displays every task for every user grouped by user. Also if the user is logged in there should be an option to show only the tasks for the current user
    • Tasks - displays a list of all tasks and which is the assigned user for the task
    • Single Task - displays all the info about a chosen task
    • Users - displays a list of all the users with the total number of tasks assigned to them.
    • Single User - displays all the info about a chosen user
  • Defining the specific route names is up to you
  • The app should use Firebase as a database
  • Users should be able to register and login with username and password.
  • Users should be able to add and change simple info about themselves, like email, first & last name, department. Usernames and password doesn’t need to be changeable.
  • Registered users should be able to create and delete tasks. Task creation doesn’t need to happen in a separate route. It can happen in a modal/popup which shows after clicking a FAB button.
  • Tasks should have the following properties: Title, Description, Assignee(dropdown with all users), Comments
  • Only registered users can delete a task or post comments.
  • Every registered user could change the task’s assignee(dropdown with all users).
  • Every name of task/user should be a link to the task/user’s page
  • All comments should be inside comment tag, which should strip out all HTML tags and leave only anchors with valid URLs
  • All comments should have Author, Comment and Posted On Datetime
  • Any unspecified conditions can be implemented as you wish
  • You may implement anything additional, that you think might be suitable

Requirements:

  • Implement at least one of every type of directive
  • Implement at least one custom filter and use it
  • Using LESS/SASS
  • Using Gulp as a build tool
  • Implement at least once a custom watch
  • Use at least once angular’s events to share data/action between controllers

Optional:

  • Using reactive programming (Rx.js)
  • Being able to visualize in a nice way a lot of data (20K tasks & 10K users)
  • Write documentation for your app explaining how it works and what’s the logic
  • As the users can only be added and not deleted. You might use localStorage to cache all the current users. So when the client comes in next time, you can initialize the users list with all the users from the last time. And in case the users request doesn’t succeed the client could still be able to see some data.
  • Implement a notification system (it’s not necessary to be real time) to show the user notifications about when he/she was assigned to a task or there was a new comment in a task to which he/she is assigned
  • Add filtering and/or sorting on lists
  • Use promises in services where asynchronous actions are required
